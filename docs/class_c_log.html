CLog Class Reference

Creates a special file stream to store logging information. More...

#include "Log.hpp"

Detailed Description

By default, one of these is created for the engine log, but more streams can be opened as needed.

The stream immediately writes to the file. If specified (via a constructor or ToggleStdout()), the data will be immediately output to the screen, as well. This eliminates the need to flush the stream repeatedly and output data in chunks, as was the case in v1.0 of IronClad.

Log output is formatted like so:

    [MODE] Subsystem – data

MODE is one of the 4 output modes in the LogMode enumerator: 

        DEBUG   –  Only shows up in debug builds (_DEBUG directive)
        INFO    –  Generic information
        ERROR   –  Error that should cause alarm but no serious harm
        FATAL   –  Critical error that terminates the program
                   with the given data as the message in the pop-up
                   window.

Logging works by calling SetMode(), which will be output for every subsequent line until SetMode() is called again. Lines are automatically detected via newline characters (std::endl won't work, avoid it). Or, they can be forced via NewLine().

You must (or should) specify the subsystem that is doing the logging via SetSystem().

You can directly stream SetMode() and SetSystem() as output, like so:

CLog Output("Output.log", false);
Output.SetMode(ZEN_LOGMODE_INFO) << Output.SetSystem("Engine")
<< "This is some data\n";
// This works too
Output << Output.SetMode(ZEN_LOGMODE_FATAL) <<
<< Output.SetSystem("Game") << "Failure.";

The output log (in Output.log) would look like this: 

    [INFO]  Engine – This is some data
    [FATAL] Game – Failure.

In addition to a popup MessageBox (on Windows) with the caption 'Game' and text saying 'Failure.'

Here are some various usage scenarios:

// Hardcoded line break.
string_t name = "halcyon";
CLog Output("Output.log", true);
Output.SetMode(ZEN_LOGMODE_INFO);
Output << "Hi " << name << ", I'm a log.";
Output.Newline();
Output << "This is the second line.";
// Automatic line break.
Output.SetSystem("Example");
Output.SetMode(ZEN_LOGMODE_DEBUG) << "Random data\n";
Output << "More data on the next line.\n";
Output.SetMode(ZEN_LOGMODE_ERROR) << Output.SetSystem("Test")
<< "And a third line."

The resulting output would look like this: 

    [INFO]  Log – Hi halcyon, I'm a log.
    [INFO]  Log – This is the second line.
    [INFO]  Example – Random data
    [INFO]  Example – More data on the next line.
    [ERROR] Test – And a third line.
