Detailed Description

A material is a combination of an effect (shaders) and a texture. It's used to specify what picture should be drawn on the screen, and how it should be drawn. There is a default instance of it in zen::gfxcore::zRenderer, and it is used for standard rendering to the screen, primarily for primitives.

The object will store a local copy of the default texture and effect, allowing for less pointer interactions and potential errors. This doesn't mean the default effect will be loaded multiple times, as the internal shaders are registered assets and asset::zAssetManager will take care of preventing copies.
