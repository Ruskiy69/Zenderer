Detailed Description

These differ from generic primitives in the sense that they have a lot more functionality abstracted away, as well as additional functions like animation and physics reactions (in inheriting classes).

They do use the zen::gfx::CQuad class at their core, but function at a much higher level.

They can also be loaded from files (see the spec) and can contain multiple primitive instances.

When creating an entity from multiple primitives, a copy of the vertex/index data will be stored internally in order to prevent a bad reference later.

The call to Optimize() is not necessary, but is recommended to merge identical primitives together if they use the same material.
